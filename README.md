An end-to-end, production-ready Machine Learning system for detecting fraudulent financial transactions. This project utilizes an advanced pipeline combining **Principal Component Analysis (PCA)**, **K-Means Clustering** for behavioral segmentation, and **XGBoost** for highly accurate classification.

The entire architecture is built using a strict, modular **MVC-like design**, exposing a **FastAPI** backend and an interactive **Streamlit** dashboard, fully containerized via **Docker**.

---

## 🏗️ Architecture

The codebase is strictly separated by responsibility to ensure maintainability and scalability:

```text
AI_final/
├── api/
│   └── main.py               # FastAPI endpoints (/predict, /update-model)
├── frontend/
│   └── app.py                # Streamlit UI for inference and batch uploads
├── models_saved/             # Pre-trained pipeline models (.pkl)
│   ├── kmeans_model.pkl
│   ├── pca_model.pkl
│   ├── scaler.pkl
│   └── xgboost_model.pkl
├── src/                      # Core ML Logic Pipeline
│   ├── continuous_learner.py # Handles partial/batch retraining
│   ├── feature_engine.py     # Applies Scaling, PCA, and K-Means segmentation
│   ├── predictor.py          # Executes XGBoost inference
│   ├── preprocessor.py       # Cleans, maps, and encodes raw data
│   └── train.py              # Training script to generate models
├── test/
│   └── generate.py           # Script to generate test batches
├── Dockerfile                # Containerization setup
├── EDA.ipynb                 # Comprehensive Data Analysis & Visualization
├── entrypoint.sh             # Startup script for FastAPI + Streamlit
└── requirements.txt          # Python dependencies
```

## ✨ Key Features

1. **Intelligent Feature Engineering**: 
   - Dynamically scales features, reduces dimensionality via PCA (retaining 98% variance), and clusters behaviors using K-Means to feed a richer dataset into the XGBoost classifier.
2. **Continuous Learning / Model Updating**: 
   - Includes a `/update-model` pipeline allowing the XGBoost model to be incrementally updated with newly labeled batch data without requiring a full retrain from scratch.
3. **Interactive UI**: 
   - A beautiful Streamlit dashboard allowing users to test single transactions in real-time or upload CSV files to trigger batch retraining.
   - Built-in data integrity constraints to ensure dynamic and realistic UI behavior.
4. **Production-Ready API**: 
   - Asynchronous FastAPI backend with strict Pydantic data validation.

## 🚀 Quick Start (Local Setup)

### 1. Prerequisites
- Python 3.10+
- The dataset `PS_20174392719_1491204439457_log.csv` placed in the root directory (ignored by git).

### 2. Installation
Clone the repository and install the dependencies:
```bash
git clone https://github.com/M-Ahmd/Fraud_Detection.git
cd Fraud_Detection
pip install -r requirements.txt
```

### 3. Generate Models
If your `models_saved/` directory is empty, run the training script to sample the dataset and generate the required pipeline models:
```bash
python -m src.train
```

### 4. Run the Application
You can run both the FastAPI backend and Streamlit frontend simultaneously using the provided bash script:
```bash
bash entrypoint.sh
```

- **Dashboard:** `http://localhost:8501`
- **API Docs (Swagger):** `http://localhost:8000/docs`

## 🐳 Docker Deployment

To run the application entirely in a Docker container without local dependencies:

```bash
# Build the image
docker build -t fraud-detection-app .

# Run the container
docker run -p 8000:8000 -p 8501:8501 fraud-detection-app
```

## 🧠 Model Pipeline Details

1. **Preprocessor (`src/preprocessor.py`)**: Extracts temporal features (Hour of day), maps account types, drops highly correlated/identifying columns, and applies One-Hot Encoding.
2. **Feature Engine (`src/feature_engine.py`)**: Applies `RobustScaler` to handle extreme monetary outliers, utilizes `PCA(n=3)` for dimensionality reduction, and appends a `behavioral_segment` generated by `KMeans(k=4)`.
3. **XGBoost Predictor**: Evaluates the processed features using an XGBoost Classifier optimized with dynamic `scale_pos_weight` to counter extreme class imbalances inherent in fraud datasets.
